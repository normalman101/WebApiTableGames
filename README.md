## Тест-план
1. Протестировать функцию в позитивном и негативном тесте.
2. В позитивном тесте использовать те данные, которые подходят для функции, исходя из её документации.
3. В отрицательном тесте использовать неподходящие данные для функции.
4. Сверить результат позитивного и отрицательного теста с ожидаемыми результатами.
5. Подробно задокументировать то, как именно проводился тест и какой результат получился.
---

## Тестирование

| Тип           | Метод  | Путь метода                | Цель                                                                                 | Ожидаемый результат                                                                      | Актуальный результат                                               | Вердикт      |
| ------------- | ------ | -------------------------- | ------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------- | ------------------------------------------------------------------ | ------------ |
| Позитивный    | GET    | /tableGames/list           | Получить списка всех игр в формате JSON                                              | Получение кода ошибки 200 со списком всех игр в формате JSON                             | Пришёл ответ с кодом ошибки 200 со списком всех игр в формате JSON | Тест пройден |
| Позитивный    | POST   | /tableGames/add            | Записать информацию о новой игре в файл                                              | Получение кода ошибки 200, обозначающий успешную запись информации в файл                | Пришёл ответ с кодом ошибки 200                                    | Тест пройден |
| Негативный    | POST   | /tableGames/add            | Попытаться записать информацию, имеющая пустые значения в полях, о новой игре в файл | Получение кода ошибки 204, обозначающую отсутствие значений у некоторых полей информации | Пришёл ответ с кодом ошибки 204 и информация не записалась в файл  | Тест пройден |
| Позитивный    | DELETE | /tableGames/delete/id/{id} | Удалить игру в файле по идентификатору                                               | Получение кода ошибки 200, обозначающую успешное удаление игры из файла                  | Пришёл ответ с кодом ошибки 200                                    | Тест пройден |
| Отрицательный | DELETE | /tableGames/delete/id/{id} | Удалить игру в файле по несуществующему идентификатору                               | Получение кода ошибки 500, обозначающую ошибку сервера                                   | Пришёл ответ с кодом ошибки 500                                    | Тест пройден |
| Позитивный    | PUT    | /tableGames/update/id/{id} | Изменить информацию о существующей игре                                              | Изменение информации игры                                                                | Пришёл ответ с кодом ошибки 200 с изменённой информацией           | Тест пройден |
| Отрицательный | PUT    | /tableGames/update/id/{id} | Изменить информацию о несуществующей игре                                            | Получение кода ошибки 500                                                                | Пришёл ответ с кодом ошибки 500                                    | Тест пройден |
